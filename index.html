Overview

GroovyServ makes Groovy's startup time quicker, by pre-invoking groovy as a server.

In the case of software development with scripting languages, it is needless to say that quick try-and-run is very important. In such situation, 2 seconds or even 1 second are too long. GroovyServ reduces sensible startup time of the JVM and Groovy runtime significantly. In most situations it is 10 to 20 times faster than regular Groovy.

Feature

GroovyServ supports following features:

  • Transparent server invocation. When the server is not running when the client is invoked, start it.
  • Connect System.in/out/err to socket stream.
  • System exit trap and exit status redirection.
  • Signal handling(Ctrl-C).
  • Change current working directry appropriately.
  • Simultaneous execution of scripts for using pipe.
  • Cygwin independence (NEW on 0.2).
  • CLASSPATH environment propergation.
  • C and ruby client.
