Overview

GroovyServ makes Groovy's startup time quicker, by pre-invoking groovy as a server.

In the case of software development with scripting languages, it is needless to say that quick try-and-run is very important. In such situation, 2 seconds or even 1 second are too long. GroovyServ reduces sensible startup time of the JVM and Groovy runtime significantly. In most situations it is 10 to 20 times faster than regular Groovy. You can see demonstration screencast.

Feature

GroovyServ supports following feature:

  • Faster invocation of groovy script(sometimes 10 times more).
  • Transparent server invocation. When the server is not running when the client is invoked, start it in background.
  • Connect System.in/out/err to socket stream.
  • Simultaneous execution of scripts. See FAQ.
  • Traps call of System.exit() and transmit the exit status to the client.
  • Signal handling(Ctrl-C).
  • Change current working directry appropriately. See FAQ.
  • Cygwin independence for windows(New feature of version 0.2).
  • CLASSPATH environment propagation.
  • C(native binary) and Ruby client.
  • Many unit tests and integration tests are added and source code is fully refactored(New feature of version 0.2).
